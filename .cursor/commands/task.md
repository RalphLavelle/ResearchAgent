# task

Find the latest task (by filename) in the /docs/tasks top-level folder (ignore 'archive' or any other non-top-level folders), and carry out any instructions in the file. For example '6.md' is later than '4.md'.

The task may be composed of several subtasks - simply do these in order.

If necessary (fundamental changes to the app, a new endpoint, etc.) update the ReadMe afer carrying out all the subtasks in the task.

Build the front end and the API back to make sure all went well.
