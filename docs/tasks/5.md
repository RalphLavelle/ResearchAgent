# Overview
A recent change was to allow the events to be filtered by venue or tag. This ha[pens by clicking on the tag or event button. The idea was that the full list that had already been fetched would just be filtered in place (NOT refetched from the database) using JavaScript, meaning it would be instantaneous and that it wouldn't involve a round-trip to the database.

## Problem
The code seems to have reverted to fetching the filtered records form the database, with the result that the filtered list is slow. This should not be happening - the records are - or should be - already available to the code. Revert this bug - make sure filtering like this is happening purely on the front-end.
