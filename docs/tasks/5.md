# New reports page
Building on the new reports endpoint, make a front-end component that shows the reports in a list at the route "/reports/"
Each report should take up one line in a table, and have columns corresponding to the property names in that collection, except for the id. On clicking anywhere on the table row, the details of the reports show in tan expended view, and any previously-expanded report row will close down again.

## Resolution

- Added `ReportsComponent` at route `/reports` with table columns `datetime`, `searches`, `urls`, `changes`.
- Rows expand on click (accordion — only one open at a time) with full detail sections.
- Nav links added (desktop + mobile). README updated.
