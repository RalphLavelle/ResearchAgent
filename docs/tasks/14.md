# Enhanced deduplication
As part of the deduplication process, make sure not to add new events that are really just the same event despite seeming to have different event/act/band names.

# Solution
In local_output.py, update the Key behaviours semantic deduplication section with the following addition.
The match on the band/event/act may be only a partial match (as in 'Singer 1' and 'Singer 1, with Singer 2', or vice versa) but if it obviously refers to the same event, i.e. the venue and date are the same then it's really a match. In this case, leave/update the act/band/event name that gives the most information as the canonical one.

Then make any necessary changes to the dedup parts of the code that reflect this.
