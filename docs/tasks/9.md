## Prompt 1 — Read-only strategy diagnostics

Implement a read-only strategy diagnostic that turns existing MongoDB memory into a concise report. It must not change pipeline behaviour.

Requirements:

- Add a testable module that reads `reports`, `sources`, and `venues`.
- Rank top remembered source URLs by past event yield.
- List venues whose `last_event_date` is missing or soonest, including `last_mined` and whether an `events_link` exists.
- List recently crawled hosts that do not appear in fruitful source memory.
- List repeated recent search queries.
- Add a script `scripts/diag_strategy.py` that prints the report for the active topic database.
- Add focused tests using the existing in-memory MongoDB fixtures.
- Verify with backend tests, frontend build, and API import check.

## Prompt 2 — Persist strategy scorecards

Add deterministic scorecard persistence after each successful run.

Requirements:

- Create a `strategy_scores` MongoDB collection or clearly named equivalent.
- Persist separate score documents for source URLs, hosts, venues, and query strings.
- Use existing run data only; do not add LLM reflection yet.
- Score should include positive signals (`events_added`, `events_seen`, duplicates) and negative signals (`zero_yield_runs`).
- Add decay-friendly fields such as `last_seen`, `runs`, and cumulative counters.
- Keep update code idempotent for one report save path.
- Add tests for score updates and API-safe serialization helpers if needed.
- Verify before continuing.

## Prompt 3 — Venue priority scoring

Use venue scores to select targeted venue queries and venue crawl seeds more intelligently.

Requirements:

- Replace purely random venue selection in targeted venue queries with weighted selection.
- Boost venues with no `events_link`, missing/soon `last_event_date`, old `last_mined`, or strong historical yield.
- Preserve exploration so low-score/new venues still get occasional coverage.
- Keep topic-specific regions in YAML; do not hard-code Brisbane or Gold Coast in engine code.
- Add deterministic tests with seeded RNG.
- Verify before continuing.
