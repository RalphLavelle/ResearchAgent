# Persistence of useful websites
At the moment, the app has no 'memory' of websites it finds that are useful and likely to yield good results. Every search starts from the same blank slate - using the prompt guide it composes search terms that are likely to yield good results, crawls the sites that result from that, saves them only in the reports collection under the urls field. But there's no mechanism to remember a url that provided any events, even though that url is likely to be fruitful in the future.

# Task
Investigate a way to record the exact URLs that, when scraped and analysed, contain events. These should be saved in a new collection, grouped by base URL, and for each new run, one of them should be randomly (although weighted slightly by how many events they have had in the past). It therefore means that after each run, that's when the URLs should be saved and/or updated with the number of events they have given us.

# Purpose
The idea here is for the app to improve as it performs its scheduled runs. If it does not learn from previous runs, it is just randomly rolling the dice each time it runs, and the app never 'learns'.

Investigate only.
