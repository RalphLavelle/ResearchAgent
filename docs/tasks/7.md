# Rework the search and pruning
Make a fundamental change in the way the app stores and saves events. The app should save ALL the events it finds in its searches and HTML parsing, even if they occur more than the nominated amount of days into the future. As a result, the searches generated should be ALL future events

## Filtering by date
The filtering by date whereby the app only shows one month's worth of events should happen only in the final query from the database. Change the api to only get events that meet that criterion, and ignore any events that fall beyond that date.

## Remove future pruning
The pruning that happens currently where events get pruned as they become out-of-date should still happen, but no 'futue-pruning' should happen.

# Be thorough
Because this change affects many parts of the app, be very thorough in the changes. Make sure everything related to the old restricted search is removed.
