# Recursive self-improvement
I have an investigation/thinking task for you to work on. How can I make the app improve so that every time the schedule runs, it 'learns' and improves over time. I mean, it can learn which search terms are likely to be fruitful, which venues are likely to have events, things like that.

Please investigate and think how the app can learn over time from past runs how to improve the efficiency of its search and any other components, in order to maximise the yield of events.

Do not make any code changes - simply save the results of the investigation into a file in the /docs/ideas folder called 'sri.md'.
