# Configurable subject matter

This is a big change - at the moment the app is made to use a certain type of subject matter, but I want it to be made easier to configure to have different subject matter. Some of this is already done, such as a swappable 'config/subject_matter.yaml' file, but most isn't and is 'hard-coded' to be about music events in Brisbane and the Gold Coast.

So, I'd like a new top-level folder called topics, with a json file at the top-level that the app reads in first thing, called 'topics.json', containing properties that can be set when a new topic is called for. This should also contain a folder per topic, containing the current top-level config files, because they will obv vary by topic.

## topic-creator skill
Create a skill responsible for creating a new topic in that object that can take a single topic name and scaffold up properties in the config file, folders and yaml files, anything necessary.

## Configurable spots
* At the moment, the data folder has everything at the top-level. It should segregate topics by folder. This folder name should be configurable.
* The web app's background image should likewise be configurable.
* The top-level 'config' folder should be moved, as discussed earlier.
* Any prompt guides/yaml prompt material that is specific to a topic should be segregated too, by topic. If they still need to live in the Python layer, in /src/agent that's fine, but they should be segregated by folder too.
