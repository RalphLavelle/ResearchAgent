# Change of search strategy
I want to make the venues more of a priority. At the moment, the search is a bit hit and miss where it comes to big venues, such as The Triffid (in Brisbane) even though that's where the majority of gigs are held. The full list of gigs at these venues is not being exploited by the agent.

## Task
When in the course of searching (the search node) what looks like a venue is found (e.g. The Triffid, https://www.thetriffid.com.au/) prioritise finding the "What's On" page for the venue, e.g. "https://www.thetriffid.com.au/whats-on" and then extracting as much as possible from those pages, bearing in mind that the results may be paged. This should be the priority of the agent - once found, a venue is 'mined' exhaustively for events.

## Save venues properties
When what looks like a venue is found, and the What's On link has been established, update the venue in the venues collection with the property (create if necessary) 'events-link' with the URL of the What's On page for that venue in order to shortcut a lookup in the future for that venue. Once found, there's no need to keep searching for a venue as if we hadn't already found it.

# Venue 'last event date'
Another property for the venue which would be very handy is the last event date, the date of the latest event crawled. This is handy because if gives us some clue as to when we need to visit that venue's events listing page again in the future.
