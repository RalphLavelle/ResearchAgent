## Completed (research shortlist — May 2026)

### How this maps to “OpenAI-compatible”

Any model you **`ollama pull`** and serve with **[Ollama](https://ollama.com/)** exposes an **OpenAI-compatible** REST API (`/v1/chat/completions`). This repo already uses that path when **`OPENAI_ENABLED=false`** (`OLLAMA_BASE_URL`, `OLLAMA_MODEL` in `.env`). So **compatibility here means: available in Ollama** — no separate API shape work.

### Reality check — your hardware (i5-7400, 32 GB RAM, Windows)

- **32 GB system RAM**: fine for typical **quantized (~Q4)** models roughly up through **13–14B**, and often **~30B weights on disk (~18–20 GB)** *if* the rest of the OS is quiet—but **generation speed on a 7‑year‑old quad‑core without a discrete GPU will drop sharply** as you go past ~8–14B. Smooth “interactive” usage usually wants a **GPU with enough VRAM**; CPU-only stays usable mainly for smaller models or patient batch runs.
- **If you have no GPU**: prioritize **same-size family upgrades first** (`qwen2.5:*`, `qwen3:8b`) before jumping to **14B+**.

Use sizes from [Ollama library pages](https://ollama.com/library) as quoted when you pull; quantized tags vary (`:latest` vs `:8b`, etc.).

### “Probably too big” for *smooth* day-to-day use on this CPU (RAM may allow, latency hurts)

Examples: **`qwen3:30b`** (~19GB quantized), **`qwen3:32b`**, **`llama3.3`** flagship sizes, **`qwen3:235b`** — doable only with **heavy patience**, excellent cooling, minimal background apps, and often better with **GPU offload**.

Model upgrades help **extraction quality** more than fixing **upstream empty search results**.
