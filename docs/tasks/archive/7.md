# Use Ollama for LLM access
I have made a new env var called OPENAI_ENABLED, and whne that's false, stop using OpenAI to get LLM access. Make a similar one, with other appropriate variables, to enable Ollama instead.

Use something like this to enable Ollama, or whatever Langgraph needs to use the model 'qwen3.5:0.8b'. This model should also be configurable. In fact, so should the server base url and the api key. 

```
import openai
client = openai.OpenAI(base_url="http://127.0.0.1:11434/v1", api_key="ollama")
response = client.chat.completions.create(
    model="qwen3.5:0.8b",
    messages=[{"role": "user", "content": "Hello"}],
    extra_body={"chat_template_kwargs": {"enable_thinking": False}}
)
```
## Documentation
Update any ReadMe or other docs to indicate that the local Ollama server has to be running for the agent to work. Previously the agent relied on cloud services from OpenAI but obviously now it has to have a local Ollama server from which it serves the models.

If the local Ollama server is not running, make sure an error to that effect is logged as soon as the agent starts.
