# Save the reports into the database
At the moment, at the end of every scheduled run, the report for that run gets saved as a markdown file into the data fodler, in the appropriate topic folder. These reports should instead be saved into the database.

## Task
Save the report at the end of the run into the collection called 'reports'. The report should have the following fields:

* datetime (UTC)
* searches (the array of searches currentl in the report under the '1. Searches' heading)
* urls (URLs the **bounded same-site crawler** actually fetched). This should be a document, whose keys are the base urls and whose values are the actual pages crawled.

* changes: The '## 4. Event store changes' part, using a dictionary instead of the rows

## API
Make an extra endpoint to expose a GET for those reports since we'll be needing to get them soon

## Tidy-up
Remove any parts of the code responsible for creating those files, since the actual md files are no longer needed.

## Resolution

- Added `report_store.py` — saves to MongoDB `reports` collection with `datetime`, `searches`, `urls`, `changes`.
- Added `GET /api/<db>/reports` (optional `?limit=` query param).
- Removed `run_report.py` and markdown file writes from `node_local_output`.
