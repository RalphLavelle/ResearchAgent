# Reorganisation

## No default queries
I've removed the fallback queries in the YAML files, so the graph_nodes should likewise get rid of the idea of defaults. If there is something wrong with the OpenAI API key for instance, the app should log an error, not fallback on defaults. Remove all instances of default_qs.

## No HTML table output, just a spreadsheet
The output of the queries ('agent_research.md') should not be a HTML table but a spreadsheet. Leave the run_log.md as it is though.

## Never overwrite
If an event is saved nto the spreadhseet, do not overwrite it on a subsequent run. Only remove events if they have expired (the date is in the past). The idea is to grow the data as subsequent runs happen, not keep resetting it.
