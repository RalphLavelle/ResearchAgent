# Modification to the 'sources memory' module
Instead of saving every source that yields an event, only save those that give MORE THAN ONE distinct event.

# Concern no.1
If I look at the saved sources (URLs) so far, a lot of them are for specific events. An example is 'https://www.songkick.com/concerts/43168849-rampage-at-fortitude-music-…' This URL, once mined for its event, has no further value. The event has been extracted and the URL has no other events.

# Solution
Only save URLs as sources if th URL yields more than 1 event.

# Concern no.2
Even if a source yields some events, it may go stale. An example of this is if the URL commits to a particula month, for example. While the 'runs_contributed' field will increment, the 'events_added' will stop incrementing.

# Solution
If the 'runs_contributed' is more than three times the 'events_added' field, instead of updating the 'last_seen' field simpley delete the url document. If that was the only one in the 'urls' field, then delete the entire source.
