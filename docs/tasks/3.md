# Responsive CSS changes
At the moment, the layout of site is not properly responsive. The columns of the main table should rearrange when the screen is below a certain width. If you look at the 'screen.png' alongside this doc, you can see the event titel, about, venue, and dates columns crsushing against one another.

# Separate the date column
Move the data colum to its own row, but only make the row when there is a change of data, similar to how it only shows the date now. Each data should be its own row, and should span the entire width of the table.

# CSS refactor
Pick sensible, normal breakpoints, save them in a variables css file, so that they can be set once and used throughout the project. Never have any pixel values anywhere other than this variables file.

# Responsive column layout
With the remaining columns: keep the image and event title separate, but move the About column's to be in the same cell as the event title, so that the event title and about are side by side. When the screen is below a certain width, move the About column to be below the event title, and make it a smaller font. 

Similarly, move the venue details underneath the 'about' part, and put it in bold

# Background image
The div or whatever html element it is that contains the 'generated...' label and the refresh button should show the background image too. At the moment it seems to have a flat white background. Make it the same transparency as the existing background, but make that rest of the page less transparent.

# 'Live music events' header
This should be the topic name, normalised (capitals where necessaary, etc) as set in the topic.json file. Remove the background transparency div and pick a cool font for it. It should be h1, and good and prominent
