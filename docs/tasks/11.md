# Venue-matching update
When an event is discovered, and the venue is decided by the parsing part of the app, a venue for the event is set. Before saving to the database, this venue is (or should be!) checked against existing venues ('venue.name' field) to avoid duplications. However, there's a danger that a venue that is, for example, 'The Imperial Hotel' will not be matched with 'Imperial Hotel'. They're clearly the same. The aliases field allows for that, but I don't want to fill it with 'The' + the venue name. Aliases are for more subtle versions of this, not just the same name missing the 'The' prefix.

# Solution
Treat the word 'The' as a special token that should be taken into consideration when searching the venues collection. It can either be present in the new venue being considered, or missing there, and vice versa with respect to the venue.name in the database. I will try to fill up the venues collection with the proper names, which will usually be the ones with the 'The' prefix. They're the ones that should show in the app.
