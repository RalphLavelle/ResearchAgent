# New search functionality
On the front page, at the top of the list of events, I'd like a searchbar that allows a user to search for events. There should be a placeholder with something like 'DJ sets on the Gold Coast', or 'Brisbane classical music'. If possible also put in an AI emoji indicating that this is AI-enabled.

On hitting enter in the search bar, the search terms should be sent to a module alongs with the events taken for the active period (30 days?) from the database to create a prompt for an LLM call. The LLM call will refine the events, and their tags, etc., to return only the ones that match the criterion.

When the searched events are loaded, the URL should be modified to append ?search={searchTerm} similar to how clicking on tags and locations modify the URL.
