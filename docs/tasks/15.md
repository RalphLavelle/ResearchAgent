# Tags
I need a bit of research please. I want the events to have 'tags' that can be used as ways of grouping different events by filtering the list to show only events that have a certain tag.

For instance, an event might be tagged 'open mic'. Similar to how the venue buttons work, users should be able to click on the open mic button to show only open mic events. Events can have up to 3 tags each.

The only real way to do this is to call on the LLM to come up with tags for each new event it find. It should prefer existing ones, but can create new ones if it needs to. So the question is, where is the best place in the whole Langgraph pipeline to generate tags using an LLM. Is it possible to include it in any of the existing LLMs, or should we start a new call?
