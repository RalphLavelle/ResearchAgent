# Investigate caching options
Each time the front page is requested, the app has to get the events that have been saved into the database, which is time-consuming.

At the end of each scheduled ru, when the new events have been saved into the database, investigate the options to cache the main (30-day) lookup so that a request from users to see the events doesn't have to hit the database each time.

What caching options are there to save looking up the database so often?
