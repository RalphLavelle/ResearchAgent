The first time the agent invokes the nominated LLM (from the .env file) any problem should result in the agent logging the error and then stopping. There's no value in going any further since the LLM will need to be used again a few times.

For instance, the agent logged this warning recently, but then kept going.

```
2026-07-08 22:00:23,590 WARNING agent.graph_nodes: Plan step failed (Error code: 404 - {'error': {'message': 'model "qwen3" not found', 'type': 'not_found_error', 'param': None, 'code': None}}) — continuing with 5 targeted venue query(ies).
```

Change the agent so it stops at this point.
