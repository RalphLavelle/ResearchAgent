# Deduplication
Given that the spreadsheet is the database, the source of truth, we want to avoid duplicate entries. So we need a way to determine whether an event is already in there, because the same event might appear in different sources.

Investigate the best place to put this code, given the existing pipeline and the proposed changes below, but it should be preventative - i.e. dupes should be prevented from happening rather than having to be dealt with after the event. 

## New column - sources
Save the website address URL into a new column called Sources. Leave the existing URL alone - that URL links to the actual gig. The sources column/property is for saving a list of alternative web locations where users can find out more about the gig. 

## Criteria
If the band name or event is the same as an existing location, with the same venue and date, then it's a duplicate entry.

## How to handle
If, according to the criteria above, it's been determined that a ptential entry is a duplicate of an existing one, save the URL of the source in the sources list for that gig, IF AND ONLY IF it's from a different domain.
